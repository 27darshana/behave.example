Important

Part of the functionality, that is described here, is experimental. It is currently not part of the official parse release. Use the jenisys/parse fork, if you want to try it out.

The parse module is the inverse of Python string.format function.

NEW FEATURES (in parse):

  • Optional pattern attribute in user-defined type-converters.

    This pattern attribute contains a regular expression for better pattern matching of the user-defined type (if present).

  • The with_pattern() decorator for type-converters.

  • parse_type extensions (TypeBuilder functionality).

  • Optional cardinality field part after type part in parse expression, like:

    Cardinality

    Example

    Description

    0..1

    “{person:Person?}”

    Zero or one: For optional parts.

    0..*

    “{persons:Person*}”

    Zero or more: For list<T> (many0).

    1..*

    “{persons:Person+}”

    One or more: For list<T> (many).
